Predicting loan outcomes

Feature Selection

For each loan we're given around 50 different features specifying the 'initial state' of the loan. Many of these are redundant, or useless, and others need to be massaged into a format that can be plugged into the models. To determine the relative importance of each loan feature we can measure how much the prediction accuracy decreases if we randomly shuffle the values of that loan feature in the test set. This analysis shows that borrower location has relatively low importance compared with some of the loan features more directly associated with a borrower's credit history.

Comparing Models

Comparing returns on portfolios of varying sizes, selected using different models, we see that even a model based only on the borrower's location can provide significantly better returns compared to chance. The full random forest model, using all loan features, can provide increases in returns of up to 10%. This analysis also shows that the number of loans in the portfolio also serves to control the bias/variance tradeoff, with larger portfolios having decreased variance (and hence lower risk), but higher bias (worse average performance).

Returns by risk level

Another important questions is the extent to which these models are improving the returns for loans across all risk levels (i.e. loan grades), or whether they are simply telling us to select from the riskier loans which have better average returns. To look at this, we can compare the portfolio returns for different models at each loan grade. Riskier loans (lower grades) do tend to provide better returns overall, but the models substantially improve returns even for loans of a given grade.

Generalizing predictions over time

Traditional K-fold cross-validation, as used above, does not tell us how well our model predictions will generalize to future loans, where the relationship between loan features and outcomes may change. One way to estimate the future performance of the model is to train it on loans issued up to a certain date, and then test its performance on predicting 'future' loan outcomes. Since the majority of loans in the LC data set were recently issued, I chose a slightly different approach, where I trained the model on ~200k loans issued in 2015 (green region below), and then tested the ability of the model to predict loans issued at varying times in the past. This ensured that there was a sufficient number of loans in the training set, while allowing for validation on test sets that were well separated in time from the training set. Somewhat surprisingly, the improved returns provided by selecting loans based on the model predictions were well-preserved when testing on loans as far back as 2010.

Modeling default probabilities

In addition to building regression models that predict the ROI of each loan directly, one can also make classifiers for predicting whether or not a loan will default. This can be useful for quantifying the risk associated with a given loan, in addition to its expected ROI. With scikit-learn this requires a bit of care (see details), but essentially I just fed the same set of predictors into a Random Forest classifier. We can then use the classifier to predict the default probabilities associated with each loan, using only the initial loan info provided, just as we would generate a predicted return. We can see that the predicted annual returns and default probabilities are negatively correlated at each loan grade (interest rate level), as we would expect.
